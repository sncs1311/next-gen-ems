2.1.2 Conditional System Business Rules (Include and Extend Contracts)

Standard Inclusions (Include Relationships)
* Log Fuel Transaction requires the user to execute View Asset Detail first.
* Create Job Card requires a valid Breakdown Report to exist in the system.
* Add Parts to Job Card requires the corresponding Job Card status to be active and open.
* Close Job Card requires mechanics to finalize and Update Job Card Status first.
* Approve Transfer requires a backend Compliance Check to pass successfully.
* Dispatch Asset requires a Pre-Departure Inspection to be fully completed.
* Confirm Arrival requires a Post-Arrival Inspection to be verified on site.
* Close Incident requires the investigator to have official Root Cause details recorded first.

Functional Extensions (Extend Relationships)
* Record Repair Warranty extends the Close Job Card — Supervisor Approval workflow. This is triggered optionally upon contract and repair completion.
* Override High-Risk Driver Assignment extends the Assign Driver to Asset workflow. This is triggered optionally by management to bypass system risk compliance warning panels.
* Enter Police Report Number extends the File Incident Report workflow. This is triggered optionally for vehicle accidents occurring on public roads.
