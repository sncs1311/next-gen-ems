TABLE 3: Asset
PURPOSE: Central master record for every piece of equipment, vehicle, or
         machinery. Every other operational entity references this table.
================================================================================

  COLUMN              | DATA TYPE      | CONSTRAINTS               | DESCRIPTION                                              | EXAMPLE
  --------------------|----------------|---------------------------|----------------------------------------------------------|-------------------
  id                  | uuid           | PK, NOT NULL              | Unique record identifier                                 | c3d4e5f6-...
  asset_number        | varchar(20)    | UNIQUE, NOT NULL          | Company-assigned number. Format: EQ-YYYY-NNNN            | EQ-2024-0001
  sub_type_id         | uuid           | FK -> AssetSubType.id     | Asset sub-type — implicitly provides category            | b2c3d4e5-...
                      |                | NOT NULL                  |                                                          |
  make                | varchar(100)   | NOT NULL                  | Manufacturer or brand name                               | Toyota
  model               | varchar(100)   | NOT NULL                  | Specific model name                                      | Land Cruiser
  year_of_manufacture | smallint       | NOT NULL                  | 4-digit year asset was manufactured                      | 2022
  ownership_type      | varchar(20)    | NOT NULL                  | Owned / Leased / Hired / Subcontractor                   | Owned
  current_status      | varchar(30)    | NOT NULL, DEFAULT Idle    | See valid statuses below                                 | Active
  current_project_id  | uuid           | FK -> Project.id, NULL    | Current project — updated by AssetSiteAssignment trigger | d4e5f6g7-...
  current_operator_id | uuid           | FK -> Driver.id, NULL     | Current driver — updated by AssetOperatorAssignment      | e5f6g7h8-...
  cumulative_hours    | decimal(10,2)  | NOT NULL, DEFAULT 0       | Total engine hours accumulated (equipment)               | 4253.50
  cumulative_km       | decimal(10,2)  | NOT NULL, DEFAULT 0       | Total kilometers accumulated (vehicles)                  | 28440.00
  color               | varchar(50)    | NULL                      | Vehicle or equipment color                               | Pearl White
  notes               | text           | NULL                      | Any additional remarks                                   | Site inspection vehicle
  is_archived         | boolean        | NOT NULL, DEFAULT false   | Soft delete — true means Written Off or archived         | false
  created_by          | uuid           | FK -> Employee.id         | User who registered this asset                           | f6g7h8i9-...
                      |                | NOT NULL                  |                                                          |
  created_at          | timestamptz    | NOT NULL, DEFAULT now()   | Record creation timestamp                                | 2024-01-15 09:00:00+03
  updated_at          | timestamptz    | NOT NULL, DEFAULT now()   | Last modification timestamp                              | 2024-06-01 14:30:00+03

VALID STATUSES:
  Active              = Deployed and operational
  Under Maintenance   = In workshop — job card open
  Idle                = Available but not assigned to a project
  In Transit          = Moving between sites — transfer dispatched
  Decommissioned      = Retired from service — awaiting write-off
  Written Off         = Formally written off — record archived

BUSINESS RULES:
  - Asset number auto-generated: EQ-[YYYY]-[0001 to 9999]
  - Cannot transition to In Transit while status is Under Maintenance
  - Cannot have fuel log or utilization log created while Written Off
  - Only Fleet Manager and System Admin may set Decommissioned or Written Off
  - Soft delete only — is_archived = true, record never physically deleted

TABLE 4: EngineSpecification
PURPOSE: All mechanical and engine-specific technical data for an asset.
         Separated from Asset because engine data is dense, rarely queried
         in list views, and an engine replacement creates a new record while
         the Asset record remains the same, preserving engine history.
================================================================================

  COLUMN                              | DATA TYPE      | CONSTRAINTS                | DESCRIPTION                                      | EXAMPLE
  ------------------------------------|----------------|----------------------------|--------------------------------------------------|-------------------
  id                                  | uuid           | PK, NOT NULL               | Unique record identifier                         | d4e5f6g7-...
  asset_id                            | uuid           | UNIQUE, FK -> Asset.id     | Parent asset — one-to-one relationship           | c3d4e5f6-...
                                      |                | NOT NULL                   |                                                  |
  engine_make                         | varchar(100)   | NOT NULL                   | Engine manufacturer brand                        | Toyota
  engine_model                        | varchar(100)   | NOT NULL                   | Engine model designation                         | 1GR-FE
  engine_serial_number                | varchar(100)   | UNIQUE, NOT NULL           | Unique serial from engine nameplate              | 1GR-FE-2022-441892
  chassis_serial_number               | varchar(100)   | UNIQUE, NOT NULL           | VIN or equipment frame serial number             | JTMHV05J504012345
  fuel_type                           | varchar(20)    | NOT NULL                   | Diesel / Petrol / Dual-Fuel / Electric           | Petrol
  displacement_liters                 | decimal(4,1)   | NULL                       | Engine displacement in liters                    | 4.0
  engine_configuration                | varchar(50)    | NULL                       | V6 / Inline-4 / V8 / Flat-4 etc.               | V6
  rated_horsepower                    | smallint       | NULL                       | Engine rated output in HP                        | 270
  rated_torque_nm                     | smallint       | NULL                       | Engine rated torque in Newton-meters             | 380
  emission_standard                   | varchar(30)    | NULL                       | Euro 3 / Euro 5 / Tier 4 Final / Stage V         | Euro 5
  transmission_type                   | varchar(30)    | NULL                       | Manual / Automatic / CVT / Torque Converter      | Automatic
  drive_configuration                 | varchar(30)    | NULL                       | 2WD / 4WD / AWD / Tracked / Wheeled              | 4WD
  cooling_system                      | varchar(20)    | NULL                       | Air-Cooled / Water-Cooled / Oil-Cooled           | Water-Cooled
  fuel_tank_capacity_liters           | decimal(6,1)   | NOT NULL                   | Tank capacity — used to validate fuel entries    | 87.0
  service_interval_oil_hours          | smallint       | NULL                       | OEM engine oil change interval in hours          | 250
  service_interval_oil_filter_hours   | smallint       | NULL                       | OEM oil filter change interval in hours          | 250
  service_interval_air_filter_hours   | smallint       | NULL                       | OEM air filter change interval in hours          | 500
  service_interval_fuel_filter_hours  | smallint       | NULL                       | OEM fuel filter change interval in hours         | 500
  service_interval_hydraulic_oil_hours| smallint       | NULL                       | OEM hydraulic oil interval — null if no hydraulics| 2000
  service_interval_greasing_hours     | smallint       | NULL                       | OEM greasing interval in hours                   | 50
  service_interval_full_inspection_hours| smallint     | NULL                       | OEM full inspection interval in hours            | 1000
  service_interval_km                 | integer        | NULL                       | Kilometer-based service interval for vehicles    | 10000
  is_current                          | boolean        | NOT NULL, DEFAULT true     | False when engine has been replaced              | true
  replaced_at                         | timestamptz    | NULL                       | Timestamp when this engine was replaced          | null
  created_at                          | timestamptz    | NOT NULL, DEFAULT now()    | Record creation timestamp                        | 2024-01-15 09:00:00+03
  updated_at                          | timestamptz    | NOT NULL, DEFAULT now()    | Last modification timestamp                      | 2024-01-15 09:00:00+03

BUSINESS RULES:
  - engine_serial_number must be globally unique — enforced by UNIQUE constraint
  - chassis_serial_number must be globally unique — enforced by UNIQUE constraint
  - fuel_tank_capacity_liters is mandatory — used by fuel log validation
  - When engine is replaced: set is_current = false on old record, create new record
  - At least one service interval field must be populated for PM schedule generation

         Separated from Asset because not all assets are road-registered.
         Heavy equipment (excavators, cranes, generators) have no plate number.
================================================================================

  COLUMN                        | DATA TYPE     | CONSTRAINTS                | DESCRIPTION                                       | EXAMPLE
  ------------------------------|---------------|----------------------------|---------------------------------------------------|-------------------
  id                            | uuid          | PK, NOT NULL               | Unique record identifier                          | e5f6g7h8-...
  asset_id                      | uuid          | UNIQUE, FK -> Asset.id     | Parent asset — one-to-one relationship            | c3d4e5f6-...
                                |               | NOT NULL                   |                                                   |
  country_of_registration       | varchar(10)   | NOT NULL                   | QAT / UAE / SAU / OMN / KWT / BHR                | QAT
  plate_number                  | varchar(30)   | UNIQUE, NOT NULL           | Official plate issued by DOT                      | QAT-21443-A
  registration_cert_number      | varchar(100)  | NOT NULL                   | Mulkiya or Istimara certificate number            | MQ-2022-88341
  registration_expiry_date      | date          | NOT NULL                   | Registration certificate expiry date              | 2024-08-22
  traffic_file_number           | varchar(100)  | NULL                       | Traffic department internal file reference        | TF-QAT-441-2022
  last_inspection_date          | date          | NULL                       | Date of last official roadworthiness inspection   | 2024-01-15
  next_inspection_due_date      | date          | NULL                       | Projected next inspection date                    | 2025-01-15
  roadworthy_certificate_number | varchar(100)  | NULL                       | Roadworthiness certificate reference              | RW-QAT-2024-00441
  export_cert_number            | varchar(100)  | NULL                       | For cross-border equipment movement clearance     | EC-QAT-2024-00112
  is_current                    | boolean       | NOT NULL, DEFAULT true     | False when registration record is superseded      | true
  created_at                    | timestamptz   | NOT NULL, DEFAULT now()    | Record creation timestamp                         | 2024-01-15 09:00:00+03
  updated_at                    | timestamptz   | NOT NULL, DEFAULT now()    | Last modification timestamp                       | 2024-01-15 09:00:00+03

BUSINESS RULES:
  - Only required for AssetSubTypes where requires_gulf_registration = true
  - System must alert at 90, 60, and 30 days before registration_expiry_date
  - Transfer dispatch is blocked if registration_expiry_date has passed
  - plate_number format validated against country_of_registration format rules
  - When renewed: create new record with is_current = true, old record is_current = false

TABLE 9: AssetInsuranceCoverage
PURPOSE: Junction table resolving the many-to-many relationship between
         Assets and InsurancePolicies. Stores the date range for which
         a specific policy covers a specific asset.
================================================================================

  COLUMN         | DATA TYPE    | CONSTRAINTS                       | DESCRIPTION                                    | EXAMPLE
  ---------------|--------------|-----------------------------------|------------------------------------------------|-------------------
  id             | uuid         | PK, NOT NULL                      | Unique record identifier                       | i9j0k1l2-...
  asset_id       | uuid         | FK -> Asset.id, NOT NULL          | Asset being covered                            | c3d4e5f6-...
  policy_id      | uuid         | FK -> InsurancePolicy.id          | Policy providing coverage                      | h8i9j0k1-...
                 |              | NOT NULL                          |                                                |
  effective_from | date         | NOT NULL                          | Date this coverage became effective            | 2024-01-01
  effective_to   | date         | NULL                              | Date coverage ended — null if currently active | null
  created_at     | timestamptz  | NOT NULL, DEFAULT now()           | Record creation timestamp                      | 2024-01-01 08:00:00+03

UNIQUE CONSTRAINT: (asset_id, policy_id, effective_from) must be unique
BUSINESS RULES:
  - To check if asset is insured: query WHERE asset_id = X AND effective_to IS NULL
  - When policy is renewed: set effective_to on old record, create new record
